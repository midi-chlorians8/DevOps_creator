Вот список основных команд Terraform, которые вы можете использовать:

**Основные команды:**
1. `init` - Подготовьте рабочий каталог для других команд.
2. `validate` - Проверьте, является ли конфигурация действительной.
3. `plan` - Показать изменения, требуемые текущей конфигурацией.
4. `apply` - Создать или обновить инфраструктуру.
5. `destroy` - Уничтожить ранее созданную инфраструктуру.

**Все остальные команды:**
1. `console` - Попробуйте выражения Terraform в интерактивном командном интерфейсе.
2. `fmt` - Отформатируйте вашу конфигурацию в стандартном стиле.
3. `force-unlock` - Снять застрявшую блокировку на текущем рабочем пространстве.
4. `get` - Установите или обновите удаленные модули Terraform.
5. `graph` - Сгенерируйте график Graphviz шагов в операции.
6. `import` - Свяжите существующую инфраструктуру с ресурсом Terraform.
7. `login` - Получите и сохраните учетные данные для удаленного хоста.
8. `logout` - Удалите локально сохраненные учетные данные для удаленного хоста.
9. `metadata` - Команды, связанные с метаданными.
10. `output` - Показать значения вывода из вашего корневого модуля.
11. `providers` - Показать поставщиков, требуемых для этой конфигурации.
12. `refresh` - Обновите состояние, чтобы соответствовать удаленным системам.
13. `show` - Показать текущее состояние или сохраненный план.
14. `state` - Расширенное управление состоянием.
15. `taint` - Пометьте экземпляр ресурса как не полностью функциональный.
16. `untaint` - Удалите состояние "загрязненности" из экземпляра ресурса.
17. `version` - Показать текущую версию Terraform.
18. `workspace` - Управление рабочим пространством.

**Глобальные опции (используйте их перед подкомандой, если таковая имеется):**
1. `-chdir=DIR` - Переключитесь на другой рабочий каталог перед выполнением данной подкоманды.
2. `-help` - Показать этот вывод справки или справку для указанной подкоманды.
